### Lịch sử hội nghị

SOMET được khởi xướng năm **2002 tại Paris** và đã hoạt động hơn 20 năm. Đây là diễn đàn uy tín trong cộng đồng Software Engineering, nơi các nhà nghiên cứu và practitioner chia sẻ kết quả nghiên cứu gốc và kinh nghiệm thực tế trong lĩnh vực software science.

### Mục tiêu

- Phản ánh state-of-the-art và xu hướng mới trong software methodologies
- Kết nối lý thuyết và thực tiễn để đề xuất giải pháp cho các bài toán Software Engineering
- Đánh giá các intelligent software techniques hiện tại
- Định hướng tương lai cho software science

## 6. Định dạng bài viết (IOS Press Format)

### 6.1 Template download

| Loại | Link |
|------|------|
| **LaTeX** (khuyên dùng) | https://github.com/vtex-soft/texsupport.IOS-Book-Article |
| **MS Word** | https://www.iospress.com/book-article-instructions |
| Hướng dẫn đầy đủ | https://www.iospress.com/book-article-instructions |

### 6.2 Quy định layout

| Thông số | Yêu cầu |
|----------|---------|
| **Font** | Times New Roman |
| **Cỡ chữ body** | 10pt |
| **Type area** | 12.4cm × 20cm (KHÔNG chỉnh tay) |
| **Line spacing** | Single (1.0) |
| **Số cột** | 1 cột |
| **Định dạng file** | PDF (submit) / LaTeX hoặc Word (camera-ready) |

### 6.3 Cấu trúc bài chuẩn

```
1. Title (viết hoa chữ đầu từ chính)
2. Authors
   - First name + Last name
   - Affiliation
   - ORCID (nếu có)
   - Email tác giả liên hệ (*)
3. Abstract (tóm tắt nội dung)
4. Keywords (từ khóa, phân cách bằng dấu chấm phẩy)
5. Introduction
6. Related Work / Background
7. Methodology / Proposed Approach
8. Experiments / Evaluation / Results
9. Discussion
10. Conclusion & Future Work
11. Acknowledgements (nếu có)
12. References
```

### 6.4 Quy định trích dẫn

**Chuẩn:** Vancouver / NLM style

- Số trong ngoặc vuông: `[1]`, `[2, 3]`, `[4–7]`
- Liệt kê theo **thứ tự xuất hiện** trong bài
- **Bắt buộc có DOI** nếu tồn tại

**Ví dụ format:**

```
Journal article:
[1] Petitti DB, Crooks VC, Buckwalter JG, Chiu V. Blood pressure levels 
    before dementia. Arch Neurol. 2005 Jan;62(1):112-6. doi: 10.xxxx/...

Conference paper:
[2] Rice AS, Farquhar-Smith WP. Cannabinoids and pain. In: Proceedings of 
    the 10th World Congress on Pain; 2002 Aug 17-22; San Diego, CA. 
    Seattle: IASP Press; 2003. p.437-68. doi: 10.xxxx/...

Book chapter:
[3] Whiteside TL, Heberman RB. Effectors of immunity. In: Kufe DW, editor. 
    Cancer medicine 6. Hamilton: BC Decker; 2003. p.221-8. doi: 10.xxxx/...

Book:
[4] Jenkins PF. Making sense of the chest x-ray. New York: Oxford 
    University Press; 2005. 194 p. doi: 10.xxxx/...
```

### 6.5 Footnotes & Endnotes
- Hạn chế tối đa footnotes
- Nếu footnotes > 10% type area → chuyển thành **Endnotes** (đặt trước References)
- Đánh số bằng chữ số Ả Rập (1, 2, 3...)

## 7. Chủ đề chấp nhận bài (Call for Papers)

### Nhóm 1: Software Engineering & Methodologies
- Requirements Engineering, đặc biệt cho high-assurance systems và requirement elicitation
- Software methodologies & tools (robust, reliable, non-fragile design)
- Software development techniques & legacy systems
- Automatic software generation vs reuse
- Software quality & process assessment for business enterprise models
- Intelligent software systems design & software evolution techniques
- **Agile Software & Lean Methods**
- Software optimization & formal methods for software design
- Static/dynamic analysis, software performance model, software maintenance
- Software reliability & diagnosis systems
- Model Driven Development (MDD) — code-centric to model-centric

### Nhóm 2: AI & Intelligent Systems ⭐
- **AI Techniques on Software Engineering & Requirement Engineering**
- Intelligent software systems design & evolution
- Ontology, cognitive models & philosophical aspects on software design
- Software design through interaction
- Precognitive software techniques for interactive/entertainment applications
- Creativity & art in software design principles
- Axiomatic-based principles on software design

### Nhóm 3: Security & Formal Methods
- Software security tools & techniques + related SE models
- Formal techniques for software representation, testing & validation
- Mobile code security tools & techniques

### Nhóm 4: Applications
- **Medical Informatics & Bioinformatics** — software methods for bio-medicine
- Emergency Management Informatics
- Civil Protection, First Response & Disaster Recovery
- End-user programming environment
- User-centered Adoption-Centric Re-engineering techniques

## 9. Mẹo để paper được chấp nhận

1. **Đọc kỹ topics** — chọn topic phù hợp nhất với nghiên cứu
2. **Đúng format ngay từ đầu** — IOS Press format, không submit rồi sửa
3. **Abstract rõ ràng** — nêu rõ problem, method, contribution, results
4. **Experimental results** — SOMET đề cao theory + practice
5. **Novelty** — phải có đóng góp mới, không chỉ survey
6. **English chuẩn** — proofread kỹ, đây là trách nhiệm của tác giả
7. **DOI đầy đủ** — tất cả references phải có DOI nếu có
8. **Submit sớm** — tránh lỗi kỹ thuật phút chót trên CMT
